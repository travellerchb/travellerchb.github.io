Linus 的7种运行级别

  • 不同的 Linux 发行版默认运行级别可能会有所不同，但较为常见的默认运行级别是 3 或 5

  • 查看系统级别

    • runlevel
      • 会显示两个数字，第一个是上一次的运行级别，第二个是当前的运行级别
    • who -r
      • 会显示当前运行级别和系统的运行状态信息
    • cat /proc/renlevel
      • 会显示与 runlevel 类似的输出结果

  • 运行级别 0

    • 系统停机状态，系统默认运行级别不能设为 0，否则不能正常启动，一开机就自动关机

  • 运行级别1

    • 单用户模式，root 权限，用于系统维护，禁止远程登陆，就像 Windows 下的安全模式登录

  • 运行级别2

    • 多用户模式，没有 NFS 网络支持

  • 运行级别3

    • 完全的多用户状态（有 NFS），登陆后进入控制台命令行模式。适用于服务器环境，系统启动全部服务但没有图形界面

  • 运行级别4

    • 在大多数 Linux 系统中是未定义的，由发行版自行决定其用途
    • 系统未使用，保留一般不用

  • 运行级别5

    • X11 控制台，登陆后进入图形 GUI 模式，适用于桌面用户使用

  • 运行级别6

    • 系统重启模式，默认运行级别不能设为 6，否则不能正常启动，就会一直开机重启开机重启

服务管理类命令

  • service 命令

    • 在一些较旧的 Linus 发行版中广泛使用，现如今支持它的发行版逐渐减少
    • service 是 init 系统下进行服务管理的命令
    • 通常直接操作服务的脚本文件来管理服务，对服务的管理相对较为简单
    • 命令
      • SCRIPT 为 /etc/init.d/ 中存放的可执行脚本文件
      • service SCRIPT start
        • 启动服务
      • service SCRIPT stop
        • 停止服务
      • service SCRIPT restart
        • 重启服务
      • service SCRIPT status
        • 查看状态服务
      • service –status-all
        • 显示所有系统服务列表
        • 其中 “+” 代表服务正在运行，而 “-” 代表服务处于关闭状态，“?” 代表根本没有状态这一说

  • chkconfig 命令

    • 在一些较旧的 Linux 发行版中常见，如 CentOS/RHEL 6 及以下版本等。随着新系统的发展，其使用逐渐减少
    • 命令
      • chkconfig
        • 查看所有服务配置
      • chkconfig 服务名 off
        • 关闭服务自启
      • chkconfig 服务名 on
        • 开启服务自启
      • chkconfig 服务名 –list
        • 查看服务开机状态

  • systemctl 命令

    • 是较新的系统和发行版中主要的服务管理工具
    • 使用 systemd 系统和单元文件来管理服务，具有更强大的服务管理机制
    • 能用systemctl管理的服务需要有一个.service文件
    • 命令
      • systemctl status
        • 显示系统状态
      • systemctl start 单元
        • 立即激活单元
      • systemctl stop 单元
        • 立即停止单元
      • systemctl restart 单元
        • 重启单元
      • systemctl enable 单元
        • 开机自动激活单元
      • systemctl disable 单元
        • 取消开机自动激活单元
      • systemctl daemon-reload
        • 重新载入 systemd，扫描新的或有变动的单元
      • systemctl is-enabled 单元
        • 查看服务是否自启
      • systemctl list-unit-files
        • 查看所有服务自启配置
      • 等等

    • 复制目录（文件夹）中的所有文件（区别于复制目录）

文件权限类命令

  • 查看权限

    • ll
      • 每个文件前端都有一个文件类型和三组 rwx
        • 文件类型
          • d 代表这是个目录（文件夹）
          • - 代表这是个文件
          • l 代表这是个链接文档
        • rwx 分别代表读、写、执行的权限，没有相应的权限用 “-” 代替
          • 没有执行的权限是进不去目录的，但可以看里面有那些文件
          • 第一组代表所有者的权限
          • 第二组代表所有组的权限
          • 第三组代表其他人的权限

  • 改变权限

    • chmod

      • 第一种变更方法

        • chmod [ugoa][+-=][rwx] 文件或目录

          • u 代表所有者

          • g 代表所有组

          • o 代表其他人

          • a 代表所有人

      • 第二中变更方法

        • chmod [0~7][0~7][0~7] 文件或目录
          • 有三种权限，用二进制表示
            • 0 代表没有权限
            • 1 代表有可执行权限
            • 2 代表有写的权限
            • 4 代表有读的权限
            • 最后将权限的值加载=在一起[0~7]，就是一组权限

  • 改变所有者

    • chown 所有者 文件或目录
    • chown 所有者:所属组 文件或目录
    • chown -R 所有者:所属组 文件或目录
      • 递归操作

  • 改变所属组

    • chgrp 最终用户组 文件或目录

搜索查找类命令

  • find
    • find 搜索范围 -name “文件或目录名”
      • 按照指定的文件或目录名查找
    • find 搜索范围 -name “*.文件后缀”
      • 模糊查找，搜索出该后缀的文件
    • find 搜索范围 -user 用户名
      • 查找搜索范围内属于该用户的文件或目录
    • find 搜索范围 -size [+-无符号]文件大小
      • + 大于
      • - 小于
      • 无符号 等于
  • grep
    • 前一个命令 | grep 查找内容
      • | 为管道符，表示将前一个命令的处理结果输出给后面的命令处理
      • 显示匹配的结果
    • 前一个命令 | grep -n 查找内容
      • 显示匹配的结果及其对应的行号
    • grep 查找内容 目标文件
      • 查找目标文件中包含指定字符的行
    • grep -v 内容
      • 反向过滤，不要显示哪些内容
  • xargs
    • 是给命令传递参数的一个过滤器，也是组合多个命令的一个工具
    • 一般是和管道一起使用

压缩和解压缩命令

  • gzip 压缩
    • 只能压缩文件，不能压缩目录
    • 不保留原来的文件
    • gzip 文件
      • 压缩为 ”文件.gz“ 文件
    • gunzip 文件.gz
      • 解压缩
  • zip 压缩
    • zip 压缩在 window 和 linus 中都通用
    • 可以保留源文件
    • zip 名字.zip 文件
      • 压缩文件
    • zip -r 名字.zip 目录
      • 压缩目录
    • unzip 名字.zip
      • 解压文件
    • unzip 名字.zip -d 指定目录
      • 解压到指定位置
  • tar 打包
    • tar -zcvf 名字.tar.gz 要打包的内容(可以多个)
      • 压缩文件
    • tar -zxvf 名字.tar.gz
      • 解压缩
    • tar -zxvf 名字.tar.gz -C 指定目录
      • 解压缩的指定路径
    • 选项
      • -z 打包同时压缩
      • -c 产生 .tar 打包文件
      • -v 显示详细信息
      • -f 指定压缩后的文件名
      • -x 解包 .tar 文件

      • 挂载前要插入光盘或者已经连接镜像文件

        • 指定文件系统的类型通常不必指定，mount 会自动选择正确的类型
      • -o options
        • 主要用来描述设备或档案的挂载方式
        • 常用参数有
          • loop
            • 用来把一个文件当成硬盘分区挂接上系统
          • ro
            • 采用只读的方式挂接设备
          • rw
            • 采用读写的方式挂接设备
          • locharset
            • 指定访问文件系统所用的字符集
    • vi /etc/fstab
      • 之后添加内容
        • 要挂接的设备 挂接的位置 文件系统的类型 defaults 0 0
      • 设置开机自动挂载
  • 卸载
    • umount 挂接的位置

系统定时任务命令

  • crond 服务管理
    • 重新启动 crond 服务
      • systemctl restart crond
  • crontab 定时任务设置
    • crontab -e
      • 编辑 crontab 定时任务
        • 通过5个位置来定义任务周期，通过空格间隔
          • 第一个位置，表示一个小时当中的第几分钟，范围为0~59
          • 第二个位置，表示一天当中的第几个小时，范围为0~23
          • 第三个位置，表示一个月当中的第几天，范围为1~31
          • 第四个位置，表示一年当中的第几月，范围为1~12
          • 第五个位置，表示一周当中的星期几，范围为0~7（0和7都代表星期日）
        • 每个 “*” 上都可以使用特殊符号
          • “*”，代表任何时间
            • 第一个位置为 “*“，代表一小时每分钟会执行一次
          • “,”，代表不连续的时间
            • 0 8,12,16 * * *，代表在每天的8点0分，12点0分，16点0分都执行一次命令
          • “-”，代表连续的时间范围
          • “*/数字”，代表每个多久执行一次
        • 注意，星期几和几号最好不要同时出现，因为它们定义的都是天，容易混乱
    • crontab -l
      • 查询 crontab 任务
    • crontab -f
      • 删除当前用户所有的 crontab 任务

Shell

基本概念

  • Shell 是一个命令行解释器，它接收应用程序/用户命令，然后调用操作系统内核
  • Shell 还是一个功能强大的编程语言，可以写脚本
  • 查看 Linus 提供的 Shell 解析器
    • cat /etc/shells
      • bash 和 sh 的关系
        • sh 是 bash 的缩写
  • 查看默认解析器
    • echo $SHELL

脚本

  • 编写
    • 编写脚本的第一句话就是指定它的解析器
      • #!/bin/bash
        • #!
          • 标记
        • /bin/bash
          • 绝对路径，告诉它用的是什么解析器
    • 接下来才写正文的内容
  • 执行
    • 第一种
      • 采用 bash 或 sh + 脚本的绝对路径或相对路径（不用赋予脚本+x权限
    • 第二种
      • 采用输入脚本的绝对路径或相对路径执行脚本（必须具有可执行权限+x）
        • 首先要赋予脚本+x权限
        • 再用路径执行脚本
    • 第三种
      • 在脚本路径前加上 “.” 或 source
    • 注意
      • 第一种执行方法，本质上是 bash 解析器帮你执行脚本，所以脚本本身不需要执行权限
      • 第二种执行方法，本质是脚本自己执行，所以需要执行权限
      • 前两种方式都是在当前 Shell 中打开一个子 Shell 来执行脚本内容，当脚本内容结束，则子 Shell 关闭，回到父 Shell 中
      • 第三种可以使脚本在当前 Shell 里运行，而无需离开子 Shell，这也是为什么我们每次修改完 /etc/profile 文件以后，需要 source 一下的原因
      • 开子 Shell 和不开子 Shell 的区别在于，环境变量的继承关系，如在子 Shell 中设置当前变量，父 Shell 是不可见的

变量

  • 系统预定义变量
    • 常用系统变量
      • $HOME、$PWD、$SHELL、$USER等
    • 查看系统变量的值
      • echo 系统变量
    • 显示当前 Shell 中所有变量
      • set
  • 自定义变量
    • 定义变量
      • 变量名=变量值，注意=前后不能有空格
    • 撤销变量
      • unset 变量名
    • 声明静态变量
      • readonly 变量，注意不能unset
    • 导出变量
      • export 变量名
        • 把变量提升为全局环境变量，可供其它 Shell 使用
    • 规则
      • 变量名称可以用字母、数字和下划线组成，但是不能以数字开头，环境变量名建议大写
      • 等号两侧不能有空格
      • 在 bash 中，变量默认类型都是字符串类型，无法直接进行数值运算
      • 变量的值如果有空格，需要使用双引号或单引号括起来
  • 特殊变量
    • $n 参数1 参数2 参数3 …
      • n 为数字，$0 代表该脚本名称，$1~$9 代表第一到第九个参数，十以上的参数需要用大括号包含，如$(10)
    • $#
      • 获取所有输入参数个数，常用与循环
    • $*
      • 代表命令行中所有的参数，$* 把所有的参数看成一个整体
    • $@
      • 代表命令行中所有的参数，不过 $@ 把每个参数区分对待
    • $?
      • 上一次执行的命令的返回状态
        • 如果值为0，证明上一个命令正确
        • 如果为非0，则证明上一个命令执行不正确

流程控制

  • 使用 Shell 脚本编辑器会简单些
  • if 判断
    • 单分支
      • if [ 条件判断式 ];then
        程序
        fi
      • if [ 条件判断式 ]
        then
        程序
        fi
    • 多分支
      • if [ 条件判断式 ]
        then
        程序
        elif [ 条件判断式 ]
        then
        程序
        else
        程序
        fi
  • case 语句
    • case $变量名 in
      “值1”)
      如果变量的值等于值1，则执行程序1
      ;;
      “值2”)
      如果变量的值等于值2，则执行程序2
      ;;
      *)
      如果变量的值都不是以上值，则执行此程序
      ;;
      esac
  • for 循环
    • for (( 初始值;循环控制条件;变量变化))
      do
      程序
      done
    • for 变量 in 值1 值2 值3 …
      do
      程序
      done
    • 注意 $* 和 $@ 的区别
  • while 循环
    • while [ 条件判断式 ]
      do
      程序
      done

正则表达式

  • 正则表达式使用单个字符串来描述、匹配一系列某个语法规则的字符串
  • 常规匹配
    • cat 文件名 | grep 字符
  • 常用特殊字符
    • ^
      • 匹配一行的开头
      • 例：cat 文件名 | grep ^a
    • $
      • 匹配一行的结尾
      • 例：cat 文件名 | grep t$
    • .
      • 匹配一个任意的字符
      • 例：cat 文件名 | grep r..t
        • 会匹配到root、rabt、rxdt等
    • *
      • 不单独使用，与上一个字符连用，表示匹配上一个字符0次会多次
      • 例：cat 文件名 | grep ro*t
        • 会匹配到rt、rot、root、rooot等
    • .*
      • 表示任意字符，任意长度
      • 例：cat 文件名 | grep r.*t
        • 会匹配到出现以r为开头，t为结尾的字符
    • [ ]
      • 表示匹配到某一范围的一个字符
        • [6,8]
          • 匹配6或8
        • [6-8]
          • 匹配一个6到8之间的数
        • [0-9]*
          • 匹配任意长度的数字字符串
        • [a-z]
          • 匹配一个a到z之间的字符
        • [a-z]*
          • 匹配任意长度的字母字符串
        • [a-c,e-f]
          • 匹配a到c或者e到f之间的任意字符
    • \
      • 转义字符，用于匹配特殊字符本身
