硬件设备与分区表示

  • 常见的硬件接口类型
    • IDE 和 SCSI
  • 在 Linus 系统中一切皆文件
    • /dev/sda
      • /dev/
        • 设备文件所在目录
      • hd：IDE设备，sd：SCSI设备，vd：虚拟设备（虚拟机中可见）
        • a：第一块 ，b：第二块，c：第三块，d：第四块
        • 一块硬盘三种分区
          • 主分区、扩展分区、逻辑分区
          • 主分区+扩展分区的编号只能1~4，逻辑分区的2编号从5开始

Linus 的7种运行级别

  • 不同的 Linux 发行版默认运行级别可能会有所不同，但较为常见的默认运行级别是 3 或 5

  • 查看系统级别

    • runlevel
      • 会显示两个数字，第一个是上一次的运行级别，第二个是当前的运行级别
    • who -r
      • 会显示当前运行级别和系统的运行状态信息
    • cat /proc/renlevel
      • 会显示与 runlevel 类似的输出结果

  • 运行级别 0

    • 系统停机状态，系统默认运行级别不能设为 0，否则不能正常启动，一开机就自动关机

  • 运行级别1

    • 单用户模式，root 权限，用于系统维护，禁止远程登陆，就像 Windows 下的安全模式登录

  • 运行级别2

    • 多用户模式，没有 NFS 网络支持

  • 运行级别3

    • 完全的多用户状态（有 NFS），登陆后进入控制台命令行模式。适用于服务器环境，系统启动全部服务但没有图形界面

  • 运行级别4

    • 在大多数 Linux 系统中是未定义的，由发行版自行决定其用途
    • 系统未使用，保留一般不用

  • 运行级别5

    • X11 控制台，登陆后进入图形 GUI 模式，适用于桌面用户使用

  • 运行级别6

    • 系统重启模式，默认运行级别不能设为 6，否则不能正常启动，就会一直开机重启开机重启

开关机类命令

  • 正确流程
    • 先数据同步，再关机或重启
  • sync 将内存中的数据同步导硬盘中
  • shutdown
    • shutdown -h 关机
      • h = halt
    • shutdown -r 重启
      • r = reboot
    • shutdown now 立刻关机
    • shutdown + 时间 等待多久后关机
  • poweroff 关闭系统
    • 相当于 shutdown -h
  • reboot 重启系统
    • 相当于 shutdown -r

服务管理类命令

  • service 命令

    • 在一些较旧的 Linus 发行版中广泛使用，现如今支持它的发行版逐渐减少
    • service 是 init 系统下进行服务管理的命令
    • 通常直接操作服务的脚本文件来管理服务，对服务的管理相对较为简单
    • 命令
      • SCRIPT 为 /etc/init.d/ 中存放的可执行脚本文件
      • service SCRIPT start 启动服务
      • service SCRIPT stop 停止服务
      • service SCRIPT restart 重启服务
      • service SCRIPT status 查看状态服务
      • service –status-all 显示所有系统服务列表
        • 其中 “+” 代表服务正在运行，而 “-” 代表服务处于关闭状态，“?” 代表根本没有状态这一说

  • chkconfig 命令

    • 在一些较旧的 Linux 发行版中常见，如 CentOS/RHEL 6 及以下版本等。随着新系统的发展，其使用逐渐减少
    • 命令
      • chkconfig 查看所有服务配置
      • chkconfig 服务名 off 关闭服务自启
      • chkconfig 服务名 on 开启服务自启
      • chkconfig 服务名 –list 查看服务开机状态

  • systemctl 命令

    • 是较新的系统和发行版中主要的服务管理工具
    • 使用 systemd 系统和单元文件来管理服务，具有更强大的服务管理机制
    • 能用systemctl管理的服务需要有一个.service文件
    • 命令
      • systemctl status 显示系统状态
      • systemctl start + 单元 立即激活单元
      • systemctl stop + 单元 立即停止单元
      • systemctl restart + 单元 重启单元
      • systemctl enable +单元 开机自动激活单元
      • systemctl disable + 单元 取消开机自动激活单元
      • systemctl daemon-reload 重新载入 systemd，扫描新的或有变动的单元
      • systemctl is-enabled + 单元 查看服务是否自启
      • systemctl list-unit-files 查看所有服务自启配置
      • 等等

文件目录类命令

  • pwd（print working directory）

    • 显示当前工作目录的绝对路径

  • ls（list）

    • 列出目录内容
    • ls -a
      • 显示全部文件，包括隐藏文件
    • ls -l（ll）
      • 列出所有文件的详细信息
        • 文件类型与权限
        • 链接数
        • 文件属主
        • 文件属组
        • 文件大小（byte）
        • 建立或最近修改时间
        • 名字

  • cd（change directory）

    • cd + 绝对路径
    • cd + 相对路径（/ 开头是根目录）
    • cd 或 cd ~
      • 直接回到家目录
    • cd ..
      • 返回上一级
    • cd -
      • 回到上一次使用的目录
    • cd /
      • 回到根目录
    • cd -P + 快捷方式
      • 路径到快捷方式下的文件
      • 如果是 cd + 快捷方式
        • 路径到快捷方式，不会进入快捷方式下的文件

  • mkdir（make directory）

    • 创建一个目录
      • mkdir + 要创建的目录
    • 创建一个多级目录（可以创建多层嵌套的目录文件夹）
      • mkdir -P + 要创建的多级目录

  • rmdir（remove directory）

    • 删除一个空目录（空文件夹）

  • touch

    • 创建空文件

  • cp

    • 复制文件
      • cp + 源文件 + 目标目录
    • 复制目录（文件夹）
      • cp -r + 源目录（文件夹） + 目标目录
    • 复制目录（文件夹）中的所有文件（区别于复制目录）
      • cp + 源目录（文件夹）* + 目标目录
    • 强制复制，覆盖原有同名文件
      • \cp + 源目录（文件夹）* + 目标目录

  • rm

    • rm -r + …
      • 递归删除目录所有内容
    • rm -f + …
      • 强制删除
    • rm -v + …
      • 显示指令的详细执行过程
      • 会显示删除了哪些文件

  • mv

    • 重命名
      • mv oldNameFile newNameFile
    • 移动文件
      • mv 文件 目标位置
    • 两者综合
      • mv oldNameFile 目标位置/newNameFile

  • 查看文件命令

    • head
      • head 文件
        • 查看文件头10行
      • head -n + 数字 + 文件（head -数字 文件）
        • 查看文件头几行
    • tail（用于查看报错日志）
      • tail 文件
        • 查看文件后10行
      • tail -n + 数字 + 文件（tail -数字 文件）
        • 查看文件后几行
      • tail -f 文件
        • 实时追踪文档的所有更新
    • 已经被淘汰的命令
      • cat
        • cat 文件
          • 查看文件内容
        • cat -n 文件
          • 显示所有行的行号，包括空行
      • more
        • more + 文件
          • 翻下一页
            • 空格键
          • 回车
            • 下翻一行
          • q
            • 退出
          • Ctrl + F
            • 向下滚动一屏
          • Ctrl + B
            • 返回上一屏
          • =
            • 输出当前行的行号
          • :f
            • 输出文件名和当前行的行号
      • less
        • less + 文件
          • 空格键
            • 向下翻一页
          • [pageup]
            • 向下翻一页
          • [pageup]
            • 向上翻一页
          • /字符串
            • 向下搜寻该字符串
            • n 向下查找
            • N 向上查找
          • ?字符串
            • 向上搜寻该字符串
            • n 向下查找
            • N 向上查找
          • q
            • 退出

  • echo

    • 打印文字信息
      • echo + 字符串（字符）
      • echo -e
        • 支持反斜杠控制的字符转换
      • \\、\n、\t、---
    • 打印环境变量
      • echo $PATH

  • > 覆盖与 >> 追加

    • ll > 文件
      • 列表中的内容写入文件中
    • ll >> 文件
      • 列表中的内容追加到文件中
    • cat 文件1 > 文件2
      • 将文件1的内容覆盖到文件2中
    • echo 内容 >> 文件
      • 将内容追加到文件中（若文件不存在，可以自动创建）

  • ln 链接

    • 硬链接
      • ln + 原文件或目录 + 链接名
    • 软链接
      • ln -s + 原文件或目录的绝对路径 + 软链接名绝对路径
      • 删除软链接
        • rm -rf + 软链接名
          • 不是 rm -rf + 软链接名/

  • history

    • 查看历史命令
      • 可以使用
        • history >> 1.txt
        • vim 1.txt
    • 也可以使用上下键来翻找

时间日期类命令

  • date +%F

  • date +%y
    • 年（后两位）
  • date +%Y
    • 年
  • date +%m
    • 月
  • date +%d
    • 日
  • date +%H
    • 小时
  • date +%M
    • 分钟
  • date +%S
    • 秒
  • date “+%Y-%m-%d %H:%M:%S”
    • 根据字符串内的格式显示时间
  • date -d “-1 day”
    • 显示日期往前一天
    • 字符串也可以是
      • “1 day ago”
      • “yesterday”
    • 后面也可以加上格式
  • date -s “具体时间”
    • 设置系统当前时间

用户管理命令

  • 添加用户
    • useradd 用户名
      • 添加新用户
    • useradd -g 组名 用户名
      • 添加用户到某个组
  • 设置密码
    • passwd 用户名
  • 查看用户
    • id 用户名
      • 查看用户是否存在
    • cat /etc/passwd
      • 查看创建的用户
  • 切换用户
    • su 用户名
      • 切换用户，只能获得用户的执行权限，不能获得环境变量
    • su - 用户名
      • 切换用户，并获得该用户的环境变量及执行权限
  • 退出用户
    • exit
  • 删除用户
    • userdel 用户名
      • 删除用户，但保存用户的主目录
    • userdel -r 用户名
      • 用户和用户主目录都删除（删除用户的同时，删除与用户相关的所有文件）
  • 显示用户
    • whoami
      • 显示自身用户名称
    • who am i
      • 显示登录用户的用户名
  • 临时 root 权限
    • sudo 命令
    • 使用该命令需要有对应的使用 sudo 的权限
      • 修改配置文件
        • vim /etc/sudoers
        • 修改 /etc/sudoers 文件，找到第101行，在 root 下面添加一行
          • 用户名 ALL=(ALL) ALL
            • 每次使用 sudo 需要输入密码
          • 用户名 ALL=(ALL) NOPASSWD:ALL
            • 不需要密码

文件权限类命令

  • 查看权限

    • ll
      • 每个文件前端都有一个文件类型和三组 rwx
        • 文件类型
          • d 代表这是个目录（文件夹）
          • - 代表这是个文件
          • l 代表这是个链接文档
        • rwx 分别代表读、写、执行的权限，没有相应的权限用 “-” 代替
          • 第一组代表所有者的权限
          • 第二组代表所有组的权限
          • 第三组代表其他人的权限

  • 改变权限

    • chmod

      • 第一种变更方法

        • chmod (ugoa)(+-=) 文件或目录

          • u 代表所有者

          • g 代表所有组

          • o 代表其他人

          • a 代表所有人

      • 第二中变更方法

        • chmod (0~7)(0~7)(0~7) 文件或目录
          • 有三种权限，用二进制表示
            • 0 代表没有权限
            • 1 代表有可执行权限
            • 2 代表有写的权限
            • 4 代表有读的权限
            • 最后将权限的值加载=在一起（0~7），就是一组权限

  • 改变所有者

    • chown 所有者 文件或目录
    • chown 所有者:所属组 文件或目录
    • chown -R 所有者:所属组 文件或目录
      • 递归操作

  • 改变所属组

    • chgrp 最终用户组 文件或目录
  • grep
    • 输出包含指定字符的行
    • grep 指定字符 目标文件
      • 查找目标文件中包含指定字符的行

编辑模式

  • 按下 i、I、o、O、a、A、r、R 等任意字母就可以进入编辑模式

  • 按 Esc 可以返回一般模式

  • i

    • 当前光标前

  • a

    • 当前光标后

  • o

    • 当前光标的下一行

  • S

  • R

    • 进入替换模式（输入替代光标字符）

命令模式

  • 输入 / 或 : 或 ? 可以进入命令模式
  • 按 Esc 可以返回一般模式

    • 保存
  • :q
    • 退出
  • :!

  • /要查找的词

    • N 往上查找
  • :noh
    • 取消高亮
  • :set nu
    • 展示行号
  • :set nonu
    • 取消行号

未保存退出

若在编辑后未先转换为命令模式再保存退出

而是使用 Ctrl + z 退出，只是类似于文件的窗口关闭，并且会生成一个隐藏文件

且无法再次打开文件

解决方法为使用 ls -a 找出隐藏文件，再使用 rm -rf 隐藏文件 来删除隐藏文件就可以解决，但编辑的内容未保存

另一种是切换为命令模式未保存强制退出，这种不会出现上述情况，但编辑内容未保存
